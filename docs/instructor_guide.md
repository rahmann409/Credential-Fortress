# Instructor Guide

## Learning Objectives
By the end of the lab sessions, students should be able to:
- Explain why common dictionary words and dates make poor passwords.
- Understand the concept of hash rates and estimated time-to-crack.
- Generate strong, high-entropy passwords that mitigate these attacks.

## Expected Outputs
- Students will generate three audit reports.
- Scenario 1 (Weak) should show near-instant estimated crack times.
- Scenario 2 (Patterns) will show the effectiveness of mutation rules (e.g., leet speak).
- Scenario 3 (Strong) will show high entropy scores and failure to overlap with basic dictionary lists.

Instructors should verify students run the tool in `--dry-run` mode first.
